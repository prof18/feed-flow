## Task 6.1: Docker-based FreshRSS Tests

### Setup

Use Docker to spin up a FreshRSS instance for testing:

```bash
docker run -d \
  --name freshrss-test \
  -p 8080:80 \
  -e TZ=America/New_York \
  freshrss/freshrss:latest
```

### Test File

**File**: `feedSync/greader/src/jvmTest/kotlin/.../e2e/FreshRSSE2ETest.kt`

```kotlin
package com.prof18.feedflow.feedsync.greader.e2e

import org.junit.jupiter.api.Tag
import kotlin.test.Test
import kotlin.test.assertTrue

@Tag("e2e")  // Only run when explicitly requested
class FreshRSSE2ETest {

    // Requires Docker container running
    private val serverUrl = "http://localhost:8080"
    private val username = "admin"
    private val password = "password"

    @Test
    fun `should complete full sync with real FreshRSS instance`() {
        // Setup: Create test account, add test feeds via API

        // 1. Login
        val client = GReaderClient(baseUrl = serverUrl)
        val authToken = client.login(username, password)
        assertTrue(authToken.isNotEmpty())

        // 2. Add a test feed
        client.addSubscription(authToken, "https://example.com/feed.xml")

        // 3. Fetch subscriptions
        val subs = client.getSubscriptions(authToken)
        assertTrue(subs.isNotEmpty())

        // 4. Fetch items
        val items = client.getStreamContents(authToken, "reading-list")
        assertTrue(items.items.isNotEmpty())

        // 5. Mark as read
        client.editTag(authToken, items.items[0].id, addTag = "read")

        // 6. Verify read status persisted
        val updatedItems = client.getStreamContents(authToken, "reading-list")
        // Verify item is marked as read

        // Cleanup: Remove test feeds
    }
}
```

### Running E2E Tests

```bash
# Start Docker container
docker-compose up -d freshrss

# Run E2E tests
./gradlew feedSync:greader:test --tests "*E2ETest" -q --console=plain

# Stop container
docker-compose down
```

**Acceptance Criteria**:
- ✅ Can spin up FreshRSS via Docker
- ✅ Full sync works with real instance
- ✅ Tests clean up after themselves

services:
  postgres:
    image: postgres:15

      POSTGRES_USER: miniflux

      POSTGRES_DB: miniflux

      - "5432:5432"

  miniflux:
    image: miniflux/miniflux:latest

## Task 6.5: Dropbox/Google Drive Tests (Fake Data Sources)

For Dropbox and Google Drive, use fake data sources in CI and manual E2E with credentials.

**File**: `feedSync/dropbox/src/commonTest/kotlin/.../DropboxSyncTest.kt`

```kotlin
package com.prof18.feedflow.feedsync.dropbox

import kotlin.test.Test

class DropboxSyncTest {

    @Test
    fun `should upload backup to Dropbox`() {
        val fakeDropbox = FakeDropboxDataSource()
        val syncer = DropboxSyncer(fakeDropbox)

        val data = "test backup data"
        syncer.uploadBackup(data)

        fakeDropbox.getLastUpload() shouldBe data
    }

    @Test
    fun `should download backup from Dropbox`() {
        val fakeDropbox = FakeDropboxDataSource()
        fakeDropbox.setupBackup("existing backup")

        val syncer = DropboxSyncer(fakeDropbox)
        val downloaded = syncer.downloadBackup()

        downloaded shouldBe "existing backup"
    }
}
```

**Manual E2E**: Run with real Dropbox credentials before release.

**Acceptance Criteria**:
- ✅ Upload/download logic tested with fake
- ✅ Manual E2E works with real Dropbox
